#### REQ-01. 미션 순차 잠금 해제


  - 미션1 제출 후 미션2 자동 해금 vs 별도 비밀번호 필요 여부

  2. 도메인 mismatch(`AccessDeniedError`) — [앱 설정 > 플랫폼 > Web > 사이트 도메인]에 `park-dundeo.github.io`를 등록했지만, 이 필드는 카카오톡 공유/로그인 리다이렉트 등에 쓰이는 설정이라 **지도 API와는 무관**했음
  3. 실제 지도 API가 보는 곳은 **[플랫폼 키] > [JavaScript 키] > [JavaScript SDK 도메인]**이라는 별도 필드 — 여기에 도메인을 등록하고서야 해결됨

  - **참고:** 카카오 디벨로퍼스 콘솔은 "플랫폼 > Web 도메인"과 "JavaScript 키 > JavaScript SDK 도메인"이 이름이 비슷해 헷갈리기 쉬움. 이후 유사 API(카카오 로그인 등) 추가 시에도 이 둘을 혼동하지 않도록 주의

- **컨셉명:** 가나안 원정대 보급품 / **핵심 메시지:** "정탐 보고 완료! 이제 이 텀블러를 들고 가나안 정복하러 가즈아!"
- **지급 조건 (순서):** ① 캠프장 도착 → ② 조별 정탐 보고 제출 → ③ 담당자/관리자 확인 → ④ 정탐 보고 완료 처리 → ⑤ 텀블러 지급

- **⚠️ 착수 전 결정 필요 — 기존 REQ-17과의 관계:**
  이 요구사항의 "정탐 보고" 항목(① 다녀온 장소 ② 기억에 남는 장면 한 가지 ③ 한 문장 보고 ④ 구호)은 `mission.html`에 이미 구현된 미션2 "정탐 보고서"(REQ-17: 우리가 본 장벽 / 우리가 발견한 믿음 / 우리의 돌파기도)와 **내용이 다름**. 게다가 기존 미션2는 "이 장소에서"(견학지 현장) 작성하는 구조인데, 이번 요청은 "캠프장 도착 후"에 하는 걸로 되어 있어 시점도 다름. 가능한 해석 두 가지:
  1. **기존 미션2를 이 스펙으로 교체** — 장벽/믿음/돌파기도 3필드 대신 장소/기억장면/한문장보고/구호 4필드로 바꾸고, 제출 시점 안내 문구도 "견학지에서"→"캠프장 도착 후"로 수정

  두 방향 모두 구현 방식이 크게 달라서(스키마 교체 vs 신규 미션 추가) **확정 필요**
- **한 문장 보고 예시(장소별, 확정 시 반영):** 전쟁기념관 "우리는 평화를 기억하는 정탐대입니다." / 서대문형무소 "우리는 자유를 기억하는 정탐대입니다." / 연세대 언더우드가 기념관 "우리는 복음의 발자국을 따라가는 정탐대입니다."
