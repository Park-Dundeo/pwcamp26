## 백로그 (우선순위 순)

### 🔴 HIGH — mission.html UX 개선

#### REQ-01. 미션 순차 잠금 해제
- **요청:** 미션이 한꺼번에 다 보이면 재미없음 → 순서대로 하나씩 해금
- **확정된 방향:**
  - 조별로 **서로 다른 비밀번호** → 다른 조 미션에 접근 어렵게
  - 비밀번호 힌트는 페이지에 표시하지 않고 **별도로 제공** (교사가 구두/카드로 전달)
- **흐름:**
  ```
  조 선택 → [🔒 비밀번호 입력] → 해금 → [미션1: 경유지 인증샷] → 제출 → [미션2: 정탐 보고서]
  ```
- **비밀번호 구성:** REQ-07 참고 (퀴즈 정답을 비밀번호로 사용)
- **미결:**
  - 비밀번호 오답 횟수 제한 여부
  - 미션1 제출 후 미션2 자동 해금 vs 별도 비밀번호 필요 여부
  - 이전 미션 다시 보기 가능 여부
- **보완 요청 (2026-07-07):** 미션1 완료 전 미션2 잠금 placeholder는 현재처럼 흐릿하게 유지하되, 문구를 더 신비롭게 변경 (예: "???", "봉인된 다음 임무" 등 — 다음 미션 내용이 미리 짐작되지 않도록)

#### REQ-09. 미션 전환 스토리텔링 연출 ✅ 완료
- **요청:** 아이들이 몰입할 수 있도록 미션1 완료 → 미션2 해금 사이에 여호수아·정탐·돌파 테마의 컨셉추얼한 스토리텔링 화면/문구 삽입
- **구현:** "MISSION CLEARED" 스탬프 + 여호수아 2:23 인용 내레이션 카드, 제출 직후 표시 후 버튼으로 다음 미션 해금. 재접속 시에는 전환 화면 생략하고 바로 미션2 표시
- **남은 결정 (선택):** 6조 내레이션을 조별로 다르게 할지 (현재는 공통 문구)

#### REQ-10. 조 선택 고정 ✅ 완료
- **요청:** 조별 비밀번호로 로그인 후에는 다른 조로 들어갈 일이 없으므로, 해금 이후 조 선택을 기기에 고정
- **구현:** 코드 입력 성공 시 조 선택 카드 숨김 → "🔒 이 기기는 N조로 고정되었어요" 배지로 대체. 새로고침/재접속해도 고정 상태 유지 (localStorage 기반)

#### REQ-11. 기기 초기화 기능 (테스트용) ✅ 완료
- **요청:** 테스트를 위해 해당 기기의 조 정보를 초기화하는 기능. 전체 초기화(진행상황+이미지 포함)와 조 정보만 초기화 두 가지로 구분
- **구현:** 페이지 하단 "⚙ 기기 초기화" 링크 → 마스터 비밀번호(`3927`) 입력 → 두 가지 선택
  - **조 정보만 초기화**: localStorage의 조 선택/잠금 상태만 삭제 (제출된 사진·보고서는 Firebase에 그대로 유지)
  - **전체 초기화**: 위 로컬 상태 + 해당 조의 Firebase 제출 데이터(미션1 사진, 미션2 보고서)까지 삭제
- **주의:** 마스터 비밀번호는 클라이언트 코드에 노출되어 있어 완전한 보안은 아님 (테스트/운영 스태프용 소프트 게이트로 설계됨)

#### REQ-12. 목적지 배너 단계적 공개 ✅ 완료
- **요청:** 조 선택/로그인하자마자 최종 목적지까지 한꺼번에 보이는 게 몰입을 해침. 미션을 깰 때마다 딱 "다음 장소"까지만 보이도록 변경
- **구현:** 배너를 단일 타겟 표시로 재설계
  - 미션1 진행 중: "1차 경유지 · 미션1" + 경유지명만 표시 (최종 목적지는 숨김)
  - 미션1 완료 → 미션2 진행 중: "다음 목적지 · 미션2" + 최종 목적지명으로 갱신
  - 미션2 완료: 배너 숨김 (오늘의 정탐 끝)

#### REQ-13. 관리자 승인 기반 미션 진행 ✅ 완료
- **요청:** 학생이 제출하면 자동으로 다음 단계로 넘어가지 않고, 관리자가 통과 여부(승인/반려)를 정한 뒤에 다음 단계로 진행되도록 변경. 미션1·미션2 둘 다 승인 필요. 반려 시 재도전 가능해야 함
- **구현:**
  - Firebase 데이터에 `status`(pending/approved/rejected), `reviewedAt`, `reviewNote` 필드 추가
  - `mission.html`: 제출 즉시 "🕐 관리자 확인 중" 대기 화면 표시. Firebase 실시간 리스너로 관리자 승인/반려를 새로고침 없이 즉시 반영
    - 승인 시: 미션1 → 스토리텔링 전환 화면 후 미션2 해금 + 목적지 배너 갱신 / 미션2 → 완료 처리 + 배너 숨김
    - 반려 시: 반려 사유(선택) 표시 + "다시 제출하기" 버튼으로 재도전 가능(사진 재촬영 / 보고서 재작성)
  - `admin.html`: 조별 상세 패널에 제출물별 상태 배지(대기/승인/반려) + "✅ 승인"/"❌ 반려" 버튼 추가. 반려 시 사유를 입력하면 학생 화면에 표시됨. 통계 카드도 검토 대기/전부 승인완료/반려됨 기준으로 재구성
- **버그 수정 (2026-07-07):** 미션1을 승인 → 반려로 되돌릴 때 이미 해금된 미션2 카드가 계속 열려있던 문제 수정. 승인이 취소되면 미션2 잠금 + 목적지 배너도 이전 단계로 복원되도록 함

#### REQ-14. 목적지 갱신 강조 + 미션 카드 내 행선지 명시 ✅ 완료
- **요청:** 모바일에서 다음 목적지가 바뀌어도 잘 눈에 띄지 않음. 또한 미션2 카드 안에는 실제 도착지 이름이 없어서(="이 장소에서"로만 표기) 어디로 가야 하는지 안 보임

#### REQ-07. 조별 비밀번호 퀴즈 6개 제작
- **요청:** 캠프 주제(돌파·여호수아·가나안)에 맞는 퀴즈 6개, 각 조 비밀번호로 사용
- **조건:**

- **초안 (확정 전 검토 필요):**

  | 조 | 퀴즈 문제 | 정답(비밀번호) | 연결 주제 |

  | 3조 | 여리고성이 무너진 날은 며칠째? (숫자) | `칠` | 7일 순종 |
  | 4조 | 이번 캠프의 주제 한 단어는? | `돌파` | 캠프 테마 |
  | 5조 | 요단강을 건넌 후 기념으로 세운 것은? | `돌` | 12돌 기념비 |

- **검토 포인트:**

  - 정답 단어 입력 방식 (초성/완성형/숫자 혼용 등)
  - 힌트 전달 방법 (구두? 카드? 미션 시작 전 공지?)

#### REQ-18. Day2 구조 변경 — 6관문 + 최종 여리고 성벽(Final) ✅ 완료
- **요청:** Day2를 "7관문 순환"이 아니라 "6관문 순환 + 전체 여리고 성벽 미션(Final)" 구조로 변경. 7번째 관문 "약속의 땅"이 기존 최종 미션과 내용상 중복되므로 순환 관문에서 빼고 전체 집합 미션으로 통합
- **최종 구성:**
  1. 오프닝 (전체 집합 · 프로그램 설명 · 조별 출발 안내)
  2. 6관문 순환: 붉은 줄의 문 / 요단강 / 무거운 짐의 언덕 / 의심의 숲 / 허영의 장터 / 침묵의 성벽
  3. 전체 여리고 성벽 미션(Final): 전 조 재집합 → 조별 결단문("우리는 ______을 돌파하고, 하나님을 따라 ______하겠습니다.") 완성 → 함께 외침 → 성벽 무너뜨리기
  4. 결단 나눔 및 마무리: 조별 결단문 한 문장 공유 · 짧은 기도 · 폐회예배 연결
- **구현:** `index.html` Day2 탭 — 관문 그리드를 6개로 줄이고 "약속의 땅" 카드를 별도 "🏆 Final — 전체 여리고 성벽 미션" 섹션으로 분리. 시간표를 오프닝(0–10) / 6관문 순환(10–70) / Final 여리고 성벽(70–80) / 결단 나눔·마무리(80–90) 구조로 갱신. "7관문" 표기가 있던 다른 탭(전체 개요, 준비 현황)도 "6관문 + Final"로 통일
- **참고:** 현재 Day2는 `mission.html`/`admin.html`에 연동된 학생 진행 추적 기능이 없고, `index.html`(교사용 정적 대시보드)에만 존재하는 콘텐츠임. 요청서의 "진행률 표시(1/6~6/6, Final)"·"관리자 화면 반영"은 실시간 앱 기능이 아니라 이 정적 안내 화면 표기로 반영함.

  - ⚠️ **미완료 — Firebase 콘솔에서 수동 조치 필요:** "관리자 외 제출 사진 전체 목록 접근 제한", "Firebase Database 접근 권한 최소화"는 Realtime Database 보안 규칙(rules) 문제라 이 저장소의 정적 페이지 코드만으로는 강제할 수 없음 (현재 admin.html의 비밀번호는 클라이언트 코드일 뿐 실제 DB 접근을 막지 못함 — API 키를 아는 사람은 규칙이 열려있으면 누구나 직접 읽기 가능). 권장 규칙 초안을 `firebase.rules.recommended.json`으로 저장소에 남겨둠 — **Firebase 콘솔 > Realtime Database > 규칙에 수동으로 붙여넣어야 실제로 적용됨**
