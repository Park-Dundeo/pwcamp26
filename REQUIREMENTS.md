## 백로그 (우선순위 순)

### 🔴 HIGH — mission.html UX 개선

#### REQ-01. 미션 순차 잠금 해제
- **요청:** 미션이 한꺼번에 다 보이면 재미없음 → 순서대로 하나씩 해금
- **확정된 방향:**
  - 조별로 **서로 다른 비밀번호** → 다른 조 미션에 접근 어렵게
  - 비밀번호 힌트는 페이지에 표시하지 않고 **별도로 제공** (교사가 구두/카드로 전달)
- **흐름:**
  ```
  조 선택 → [🔒 비밀번호 입력] → 해금 → [미션1: 경유지 인증샷] → 제출 → [미션2: 정탐 보고서]
  ```
- **비밀번호 구성:** REQ-07 참고 (퀴즈 정답을 비밀번호로 사용)
- **미결:**
  - 비밀번호 오답 횟수 제한 여부
  - 미션1 제출 후 미션2 자동 해금 vs 별도 비밀번호 필요 여부
  - 이전 미션 다시 보기 가능 여부
- **보완 요청 (2026-07-07):** 미션1 완료 전 미션2 잠금 placeholder는 현재처럼 흐릿하게 유지하되, 문구를 더 신비롭게 변경 (예: "???", "봉인된 다음 임무" 등 — 다음 미션 내용이 미리 짐작되지 않도록)

#### REQ-09. 미션 전환 스토리텔링 연출 ✅ 완료
- **요청:** 아이들이 몰입할 수 있도록 미션1 완료 → 미션2 해금 사이에 여호수아·정탐·돌파 테마의 컨셉추얼한 스토리텔링 화면/문구 삽입
- **구현:** "MISSION CLEARED" 스탬프 + 여호수아 2:23 인용 내레이션 카드, 제출 직후 표시 후 버튼으로 다음 미션 해금. 재접속 시에는 전환 화면 생략하고 바로 미션2 표시
- **남은 결정 (선택):** 6조 내레이션을 조별로 다르게 할지 (현재는 공통 문구)

#### REQ-10. 조 선택 고정 ✅ 완료
- **요청:** 조별 비밀번호로 로그인 후에는 다른 조로 들어갈 일이 없으므로, 해금 이후 조 선택을 기기에 고정
- **구현:** 코드 입력 성공 시 조 선택 카드 숨김 → "🔒 이 기기는 N조로 고정되었어요" 배지로 대체. 새로고침/재접속해도 고정 상태 유지 (localStorage 기반)

#### REQ-11. 기기 초기화 기능 (테스트용) ✅ 완료
- **요청:** 테스트를 위해 해당 기기의 조 정보를 초기화하는 기능. 전체 초기화(진행상황+이미지 포함)와 조 정보만 초기화 두 가지로 구분
- **구현:** 페이지 하단 "⚙ 기기 초기화" 링크 → 마스터 비밀번호(`3927`) 입력 → 두 가지 선택
  - **조 정보만 초기화**: localStorage의 조 선택/잠금 상태만 삭제 (제출된 사진·보고서는 Firebase에 그대로 유지)
  - **전체 초기화**: 위 로컬 상태 + 해당 조의 Firebase 제출 데이터(미션1 사진, 미션2 보고서)까지 삭제
- **주의:** 마스터 비밀번호는 클라이언트 코드에 노출되어 있어 완전한 보안은 아님 (테스트/운영 스태프용 소프트 게이트로 설계됨)

#### REQ-12. 목적지 배너 단계적 공개 ✅ 완료
- **요청:** 조 선택/로그인하자마자 최종 목적지까지 한꺼번에 보이는 게 몰입을 해침. 미션을 깰 때마다 딱 "다음 장소"까지만 보이도록 변경
- **구현:** 배너를 단일 타겟 표시로 재설계
  - 미션1 진행 중: "1차 경유지 · 미션1" + 경유지명만 표시 (최종 목적지는 숨김)
  - 미션1 완료 → 미션2 진행 중: "다음 목적지 · 미션2" + 최종 목적지명으로 갱신
  - 미션2 완료: 배너 숨김 (오늘의 정탐 끝)

#### REQ-13. 관리자 승인 기반 미션 진행 ✅ 완료
- **요청:** 학생이 제출하면 자동으로 다음 단계로 넘어가지 않고, 관리자가 통과 여부(승인/반려)를 정한 뒤에 다음 단계로 진행되도록 변경. 미션1·미션2 둘 다 승인 필요. 반려 시 재도전 가능해야 함
- **구현:**
  - Firebase 데이터에 `status`(pending/approved/rejected), `reviewedAt`, `reviewNote` 필드 추가
  - `mission.html`: 제출 즉시 "🕐 관리자 확인 중" 대기 화면 표시. Firebase 실시간 리스너로 관리자 승인/반려를 새로고침 없이 즉시 반영
    - 승인 시: 미션1 → 스토리텔링 전환 화면 후 미션2 해금 + 목적지 배너 갱신 / 미션2 → 완료 처리 + 배너 숨김
    - 반려 시: 반려 사유(선택) 표시 + "다시 제출하기" 버튼으로 재도전 가능(사진 재촬영 / 보고서 재작성)
  - `admin.html`: 조별 상세 패널에 제출물별 상태 배지(대기/승인/반려) + "✅ 승인"/"❌ 반려" 버튼 추가. 반려 시 사유를 입력하면 학생 화면에 표시됨. 통계 카드도 검토 대기/전부 승인완료/반려됨 기준으로 재구성
- **버그 수정 (2026-07-07):** 미션1을 승인 → 반려로 되돌릴 때 이미 해금된 미션2 카드가 계속 열려있던 문제 수정. 승인이 취소되면 미션2 잠금 + 목적지 배너도 이전 단계로 복원되도록 함

#### REQ-14. 목적지 갱신 강조 + 미션 카드 내 행선지 명시 ✅ 완료
- **요청:** 모바일에서 다음 목적지가 바뀌어도 잘 눈에 띄지 않음. 또한 미션2 카드 안에는 실제 도착지 이름이 없어서(="이 장소에서"로만 표기) 어디로 가야 하는지 안 보임
- **구현:**
  - 목적지 배너: 실제로 단계가 바뀔 때(예: 경유지→최종 목적지)만 펄스 링 애니메이션 + 목적지 글자 팝 애니메이션 + "🔔 목적지 갱신!" 태그를 잠깐 표시해 변경을 강조
  - 미션1 카드에 "📍 경유지: OOO" 칩, 미션2 카드에 "🏁 도착지: OOO" 칩을 추가하고 안내 문구의 "이 장소에서"도 실제 도착지명으로 교체 — 배너를 안 봐도 미션 카드 안에서 바로 어디로 가야 하는지 확인 가능

- **역할:** ① 준비 중인 내용 공유 ② 확정 vs 준비중 구분 표시 ③ 선생님이 댓글로 의견·질문·보완점 제시 ④ 프로그램팀이 확인 후 필요 시 본문에 반영 ⑤ 댓글은 공식 결정사항이 아닌 의견 수렴용
- **화면 구성:**
  - PC: 우측 사이드 패널, "준비 현황" 버튼으로 열림
  - 모바일: 하단 플로팅 버튼 → 전체화면 또는 하단 모달
- **준비 소식 데이터 구조:** 제목 / 내용 / 관련 구분 / 상태 / 업데이트일 / 작성자 / 댓글 허용 여부
  - 관련 구분: 전체 · Day1 · Day2 · 웹 · 장소 · 식사 · 우천 대체 · 기타
  - 상태: 준비 중 · 의견 필요 · 확정 · 완료 · 보류
- **댓글(의견) 기능:**
  - 입력 항목: 이름 / 의견 내용 / 관련 소식 선택 / 작성 시간(자동)
  - 운영 원칙: 자유게시판이 아닌 **의견함**으로 운영 — 댓글은 결정사항 아님, 반영 여부는 프로그램팀이 판단, 반영 시 "최근 업데이트"에 기록 가능, 부적절한 댓글은 관리자가 삭제
  - **접근 제한 (1차 구현 권장안):** 댓글 작성 시 이름 + "선생님 코드"(공통 비밀번호) 입력 필요 — 링크만 아는 외부인의 무분별한 작성 방지
- **최근 업데이트 섹션:** 준비 소식 본문이 언제 어떻게 바뀌었는지 날짜별로 보여주는 로그 (예: "7/8 — Day2 구조를 6관문+Final로 변경")
- **관리자 기능 (`admin.html`에 추가):**
  - 준비 소식: 추가 / 수정 / 상태 변경 / 삭제·숨김
  - 댓글: 목록 확인 / 삭제 / 확인 처리 / "본문에 반영함" 표시
- **데이터 저장 방식:**
  - 1차 구현: 준비 소식은 정적 데이터(코드 내 배열), 댓글만 Firebase Realtime DB에 저장
  - 2차 구현(추후): 준비 소식도 관리자 페이지에서 추가/수정 가능하도록 Firebase 기반으로 확장
- **초기 준비 소식 데이터 (6건, 1차 구현 시 정적으로 삽입):**
  1. Day1 가나안 정탐 장소 확인 중 — 전쟁기념관/서대문형무소/양화진 관람 가능 여부·예약·비용·관람 시간 확인 중 (상태: 준비 중 / Day1·장소)
  2. Day1 중간 경유지 및 인증 포인트 준비 중 — 장소별 경유지 2개·인증 포인트 준비 중 (상태: 준비 중 / Day1·미션)
  3. Day1 조별 식사 운영 방식 검토 중 — 자유식사 기본, 추천 식당 후보 준비 예정 (상태: 의견 필요 / Day1·식사)
  4. Day2 돌파의 순례길 구조 정리 중 — 6관문 순환 + 전체 여리고 성벽 미션으로 정리 중 (상태: 준비 중 / Day2·프로그램) — REQ-18과 연동, 이미 확정된 부분은 상태를 "확정"으로 갱신 필요
  5. Day2 부스 게임 아이디어 수집 중 — 부스 게임·우천 대체 게임 아이디어 수집 중 (상태: 의견 필요 / Day2·부스)
  6. 웹 미션 제출 방식 준비 중 — 조별 인증사진·정탐 보고서 제출 시스템 준비, 개인정보 안내 문구 반영 예정 (상태: 준비 중 / 웹·미션) — REQ-13/17/19와 연동, 완료된 부분은 상태 "완료"로 갱신 필요
- **구현 및 미결 사항에 대한 결정:**
  - **통합 위치 → 별도 `news.html`로 분리 확정.** index.html은 기존 컨벤션대로 "읽기 전용 대시보드"로 유지하고, 쓰기 기능(댓글)은 다른 페이지들처럼 별도 파일로 분리. index.html 헤더에 "💬 준비 소식" 링크 추가
  - **"선생님 코드" → 1차 구현은 `admin.html`의 `ADMIN_PW`(`pwave2026`)를 그대로 재사용.** 새 비밀번호를 따로 관리하지 않기 위한 선택 — 나중에 분리하고 싶으면 `news.html`의 `TEACHER_CODE` 상수만 바꾸면 됨
  - **초기 데이터 6건의 상태값 → 실제 진행 상황에 맞게 갱신함:** 4번(Day2 구조)은 "확정", 6번(웹 미션 제출)은 REQ-13/17/19 완료를 반영해 "완료"로 표시. 나머지 4건은 원문 그대로 "준비 중"/"의견 필요" 유지
  - **댓글 삭제 → 완전 삭제로 구현.** admin.html에 "🗑 삭제"(영구 삭제)와 "✅ 반영함"(resolved 플래그, 취소 가능) 두 버튼을 분리 제공 — 소프트 삭제 대신 반영 여부 표시로 이력 관리
  - **댓글 작성 접근 제한:** 이름+의견 입력 후 제출 시 선생님 코드 모달이 뜨고, 1회 인증하면 세션 동안(`sessionStorage`) 재입력 없이 계속 작성 가능
- **admin.html 모더레이션:** "💬 준비 소식 의견함" 섹션 추가 — 전체 뉴스 항목의 댓글을 시간순으로 모아 보여주고, 항목별로 소식 제목 배지 + 반영/삭제 버튼 제공
